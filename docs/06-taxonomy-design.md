# Taxonomy Design

Objectives

- Define controlled vocabularies and classification schemes for assets and documentation.

Scope

- Asset classes, document types, project phases, region tags, and regulatory tags.

Stakeholders

- IA, Asset Manager, Knowledge Manager.

Deliverables

- Taxonomy matrix, authority list, tag policy.

Processes

- Taxonomy change requests, regular pruning and validation.

Governance Mechanisms

- Taxonomy owner and versioning policy.

Templates

- Tag usage examples and common tag lists.

Metrics

- Tag coverage, percentage of documents tagged correctly.

Risks

- Tag proliferation causing inconsistent usage.

Mitigation Strategies

- Controlled vocabularies, dropdowns in authoring UI.

Best Practices

- Keep tag lists small, provide clear tag definitions.
