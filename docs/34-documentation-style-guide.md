# Documentation Style Guide

Objectives

- Provide a comprehensive style guide aligned to UK English and NZ accessibility guidance.

Scope

- Tone, spelling, headings, code blocks, diagrams, captions, and accessibility.

Stakeholders

- Technical Writers, Editors, Accessibility Lead.

Deliverables

- Style guide with examples, allowed terminology list, and forbidden usages.

Processes

- Style reviews during editorial stage; automated linters for common checks.

Governance Mechanisms

- Style owner and style exceptions process.

Templates

- Style snippets and common phrasing templates.

Metrics

- Linting pass rate, number of style exceptions.

Risks

- Inconsistent tone across content.

Mitigation Strategies

- Training and enforced linters in CI.

Best Practices

- Use active voice, keep sentences short, and prefer plain language.
